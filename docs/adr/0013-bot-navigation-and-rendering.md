# 0013. Навигационный контекст в callback_data и рендеринг новым сообщением

Статус: Accepted

## Контекст

Слой `bot` строит inline-навигацию категория→подкатегория→страница товаров→карточка плюс поиск.
Две связанные проблемы:

1. **Кнопка «Назад» нуждается в контексте.** Чтобы с карточки вернуться на ту же страницу списка,
   нужны id подкатегории и номер страницы; чтобы со страницы списка вернуться к подкатегориям — id
   категории. `CatalogService` не даёт обратных карт (`sub_id → cat_id`), поэтому контекст надо
   переносить самим. При этом `callback_data` ≤ 64 байта и в нём запрещены строки/UZ-названия/запрос
   (CLAUDE.md).
2. **Как обновлять экран** на нажатие кнопки: редактировать текущее сообщение (`edit_text`) или слать
   новое (`answer`/`answer_photo`).

## Решение

- **Контекст «назад» — в самом `callback_data`** короткими 12-hex `group_id` + номер страницы:
  `pg:<cat>:<sub>:<page>`, `p:<prod>:<cat>:<sub>:<page>`. Длина id фиксирована, поэтому худший пакет
  (`p:` + 4 поля) укладывается в 64 байта (тест `test_worst_case_within_64_bytes`).
- **Поиск** держит запрос в FSM-data, а в callback кладёт только номер страницы (`sp:<page>`) и id
  товара (`ps:<prod>`); возврат с карточки поиска — на текущую страницу результатов из FSM-data.
- **Экраны рендерятся новым сообщением** (`answer`/`answer_photo`), без `edit_text`.

## Последствия

- Навигация без серверного состояния стека: любой шаг самодостаточен и переживает рестарт процесса
  (кроме поиска, чей запрос — в in-memory FSM; при рестарте сбрасывается, как и язык).
- `edit_text` несовместим с карточкой-фото (нельзя превратить текстовое сообщение в фото и обратно);
  «send-new» единообразен, прост и не ловит `message is not modified`.
- Чат наполняется сообщениями при навигации (минус UX). ДОПУЩЕНИЕ: приемлемо для прайс-листа; TODO —
  перейти на `edit` для list-экранов (без фото) при необходимости.

## Альтернативы (отвергнуты)

- **FSM-стек навигации** (хранить путь в состоянии): сложнее, требует persist для устойчивости к
  рестарту, не нужен — контекста в callback достаточно.
- **Редактирование сообщения (`edit_text`/`edit_caption`)**: ломается на переходах текст↔фото, даёт
  ошибки «not modified», усложняет тесты; выгода (чистый чат) не перевешивает.
